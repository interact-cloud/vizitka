# Telegram Mini App «Лёгкий старт» — техническая карта

Мини-приложение (TMA) для нового Бизнес-Партнёра Siberian Wellness: справочный хаб по программам, подбору продуктов, возражениям и FAQ + связь с живым наставником. Собрано на основе [../brief.md](../brief.md) и [../sw_research.md](../sw_research.md).

> Стек: чистый HTML + CSS + JavaScript, без фреймворков. Telegram Web App SDK подключён в `index.html`.

---

## Файлы — кто за что отвечает

| Файл | Ответственность |
|---|---|
| `index.html` | Точка входа. Подключает SDK, стили, `data.js`, `app.js`. Каркас: шапка + контейнер `#screens`. |
| `data.js` | **ВСЕ ДАННЫЕ** — прогресс, программы, цели/продукты, возражения, FAQ, контакт наставника. Правки контента — только здесь. |
| `app.js` | Логика: инициализация Telegram (тема, имя, кнопка «назад»), навигация между экранами, рендер экранов из `DATA`. |
| `styles.css` | Стили: нативная тема Telegram (светлая/тёмная через переменные), мобильный формат, анимации. |
| `CLAUDE.md` | Этот файл. |

## Навигация между экранами

Все экраны — это функции в объекте `SCREENS` (в `app.js`). Активный экран рендерится в `<main id="screens">`. История переходов — массив `history`, кнопка «назад» возвращает на предыдущий.

```
home (главное меню / хаб)
├── today         — Задачи на сегодня (афоризм + 3 задачи дня цикла 1–30, по кругу; галочки в localStorage)
├── progress      — Мой план на месяц (автономный калькулятор: считает план до конца месяца по 3 программам, партнёр сам отмечает баллы, всё в localStorage)
├── programs      — Список 3 программ
│   └── program:<id>   — Детали программы (club200 / club1000 / loyalty)
├── goals         — Подбор продукта: выбор цели
│   └── goal:<id>      — Продукты под цель (energy, sleep, immunity, ...)
├── objections    — Возражения (аккордеон)
├── faq           — Частые вопросы (аккордеон)
└── mentor        — Связь с Евгенией (handoff)
```

**Как переходить:** `nav('home')`, `nav('programs')` или составной маршрут с аргументом — `nav('program:club200')`, `nav('goal:energy')`. Функция `nav()` добавляет виброотклик и пишет в историю.

**Кнопка «назад»:** работает и в нашей шапке (`#backBtn`), и нативная Telegram (`tg.BackButton`). На главном экране скрыта.

## Тема и фирменный стиль

Приложение оформлено в **фирменном стиле Siberian Wellness**, а не в нативной теме Telegram. Цвета из Telegram НЕ копируются.

- **Палитра** — в `styles.css`, блок `:root`: сиреневый фон `#c9c2ea`, фиолет `#6c5ce7` (кнопки/акцент), бирюза `#1fb8ae`, золото `#e6c089`, текст `#2d2a5a`.
- **Шрифт** — Montserrat (подключён в `index.html` через Google Fonts).
- **Светлая/тёмная тема:** `app.js` → `applyTheme()` берёт у Telegram только признак тёмной темы (`tg.colorScheme`) и ставит класс `html.theme-dark`. В тёмной теме — тёмный фон, но те же фирменные акценты. Вне Telegram признак берётся из системной темы браузера (`prefers-color-scheme`).
- Цвета программ (бирюза/фиолет/оранж) заданы в `data.js` у каждой программы и применяются инлайн.

## Как открыть для проверки

- **В браузере:** открыть `index.html` — работает с темой по умолчанию (без данных пользователя Telegram). Удобно для проверки вёрстки. Для имитации мобильного — режим устройства в DevTools (ширина 320–430px).
- **В Telegram:** залить файлы на хостинг (HTTPS обязателен), создать Mini App через @BotFather, указать URL. Только так работают реальная тема, имя пользователя и кнопка «назад».

## Чего здесь намеренно НЕТ

- **Онбординг «по дням»** (День 0/1/3…) и **ежедневный цикл** — это работа самого Telegram-бота (рассылки по расписанию), а не Mini App. В приложении — справочная/хаб-часть. См. [../sw_onboarding_bot.md](../sw_onboarding_bot.md) и [../sw_daily.md](../sw_daily.md).
- **Синхронизация с ЛК/CRM не нужна** — экран «Мой план» работает автономно: партнёр сам отмечает баллы, приложение считает план на устройстве (localStorage). Владелец обновляет только цены/продукты в `data.js`. Реальные платежи — на стороне сайта компании.
