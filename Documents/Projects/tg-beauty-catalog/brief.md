# Brief: Telegram Mini App — каталог услуг бьюти-мастера (MVP v1)

> Основа: [research.md](research.md). Этот файл — план первой версии. Только то, что идёт в релиз.

---

## Общие принципы (для всех экранов)

- **Платформа:** Telegram Mini App, открывается из бота / inline-кнопки / прямой ссылки `t.me/<bot>/app`.
- **Дизайн:** тёмный премиум + золото (см. `CLAUDE.md`). Системную тему Telegram **игнорируем** — фиксированная палитра.
- **Шрифты:** Cormorant Garamond (заголовки, имена, цены), Montserrat 300/400/500 (всё остальное).
- **Нативность Telegram:**
  - `WebApp.ready()` + `WebApp.expand()` при загрузке.
  - **MainButton** — главный CTA каждого экрана (внизу, золотая `#c9a84c`, текст по контексту).
  - **BackButton** — на всех экранах, кроме главного.
  - **HapticFeedback** — `selectionChanged()` на выборе слота, `impactOccurred('light')` на тапах по карточкам, `notificationOccurred('success'/'error')` на результатах.
  - `showAlert` / `showConfirm` / `showPopup` вместо `alert()`/`confirm()`.
  - `<meta name="theme-color" content="#080b12">` + тёмный фон body инлайн-стилем (чтобы не было светлого фрейма на старте).
- **Кнопки:** минимум **44×44 px** тач-зона. Между кнопками ≥ 8 px.
- **Текст:** минимум. Заголовки экранов 1 строкой. Микротексты подсказок — только где без них непонятно.
- **Язык:** русский, обращение на «ты», живой (по `CLAUDE.md`).
- **Загрузка:** skeleton-карточки, lazy-load фото (`loading="lazy"`, WebP/AVIF), бандл цель < 200 КБ gzipped.
- **Авторизация:** через `initData` (HMAC-валидация на бэке + проверка `auth_date`). Имя берём из `initData.user.first_name`. Никаких логинов.
- **Safe-area:** `env(safe-area-inset-bottom)` для всех нижних блоков.

## Экран 1. Главная

**Что видит пользователь:**
- Hero-блок: фото мастера на весь экран (или верхние 60%), сверху мягкий градиент к фону.
- Имя мастера крупно (Cormorant 32–40 px, золотом).
- Подзаголовок: специализация одной строкой (Montserrat 14 px, `--muted`).
- Приветствие по имени из `initData`: «Привет, Анна» (Cormorant 20 px) — если `initData` есть.
- Социальный proof одной строкой: «312 записей · 4.9 ★» (Montserrat 13 px).
- 3 быстрых тайла-категории (плитки 2 строки × 1.5, фото-превью + название): «Маникюр», «Педикюр», «Уход».
- Внизу — текстовые ссылки: «Мои записи» · «О мастере» (16 px, разделены точкой).

**Действия:**
- Тап по тайлу категории → каталог с выбранной категорией.
- Тап по «Мои записи» → экран 7.
- Тап по «О мастере» → экран 8.
- **MainButton:** «Смотреть все услуги» → экран 2 (каталог, все категории).

**Чего нет:** портфолио-сторис, баннеры акций, поиск.

## Экран 2. Каталог услуг

**Что видит:**
- Заголовок «Услуги» (Cormorant 24 px).
- Горизонтальный скролл чипов-категорий вверху: «Все», «Маникюр», «Педикюр», «Уход». Активный — золотой фон, остальные — обводка `--border`.
- Вертикальный список карточек услуг:
  - Фото-превью 80×80 (слева, скруглённое 12 px).
  - Название услуги (Montserrat 15 px, `--text`).
  - Длительность · цена «от» одной строкой: «60 мин · от 2 500 ₽» (Montserrat 13 px, `--muted`, цена золотом).
  - Справа — шеврон `›`.
- Skeleton-карточки во время загрузки.

**Действия:**
- Тап по чипу → фильтр списка, haptic `selectionChanged`.
- Тап по карточке → экран 3, haptic `impactOccurred('light')`.
- **BackButton** → главная.
- **MainButton** скрыта на этом экране.

**Чего нет:** поиск, сортировка, фильтр по цене, корзина из нескольких услуг.

## Экран 4. Календарь / выбор времени

**Что видит:**
- Заголовок «Когда удобно» (Cormorant 22 px).
- Сводка вверху одной строкой: «Маникюр · 60 мин · от 2 500 ₽» (Montserrat 13 px, `--muted`).
- Горизонтальный скролл дней — 14 карточек 56×72 px:
  - День недели (Пн, Вт…) сверху (12 px).
  - Число крупно (20 px).
  - Месяц мелко (10 px, `--muted`).
  - Активный день — золотой фон + тёмный текст. Прошедшие — серые, не кликаются.
- Сетка слотов 30-минутных кнопок, 3 в ряд, высота 48 px:
  - Свободный — обводка `--border`, текст `--text`.
  - Занятый — `opacity: 0.3`, `pointer-events: none`, перечёркнут визуально.
  - Выбранный — золотой фон.
- Если на выбранный день слотов нет — плейсхолдер «На этот день всё занято. Посмотри другой».

**Действия:**
- Тап по дню → перерисовать сетку, haptic `selectionChanged`.
- Тап по слоту → выбран, haptic `selectionChanged`, активирует MainButton.
- **BackButton** → карточка услуги.
- **MainButton:** «Продолжить» (становится активной после выбора слота) → экран 5.

**Чего нет:** выбор мастера (он один), несколько слотов сразу, «ближайшее свободное» автоматом.

## Экран 5. Подтверждение записи

**Что видит:**
- Заголовок «Подтверди запись» (Cormorant 22 px).
- Карточка сводки (фон `--surface`, padding 20 px, скругление 16 px):
  - Услуга: «Маникюр» (Montserrat 16 px).
  - Дата и время: «Чт, 28 мая · 14:30» (Montserrat 15 px, золотом).
  - Длительность: «60 мин» (Montserrat 13 px, `--muted`).
  - Цена: «от 2 500 ₽» (Cormorant 20 px, золотом).
- Имя клиента из `initData` (поле read-only, можно отредактировать тапом).
- Кнопка-плашка «📞 Поделиться номером» (вызывает `requestContact()`) — показывается, если телефон ещё не получен.
- Текстовое поле «Комментарий мастеру» (опционально, placeholder «Если есть пожелания», 1 строка, expand при фокусе).
- Микротекст: «Оплата на месте. Мастер подтвердит запись в течение часа» (Montserrat 12 px, `--muted`).

**Действия:**
- Тап «Поделиться номером» → нативный `requestContact` Telegram.
- Тап по комментарию → клавиатура, viewport сжимается, нижняя плашка остаётся над клавиатурой через `viewportStableHeight`.
- **BackButton** → календарь.
- **MainButton:** «Записаться» (активна, когда есть телефон) → отправка на бэк, `showProgress()`, после ответа → экран 6 + haptic `success`. При ошибке — `showAlert`, haptic `error`.

**Чего нет:** оплата онлайн / Telegram Stars, выбор способа оплаты, промокод, согласие на рассылку.

## Что НЕ войдёт в v1 (явно отложено)

| Фича | Почему отложили |
|------|-----------------|
| **Telegram Stars / предоплата** | Регуляторный риск (Apple/Google могут зарубить «оплату визита»), плюс лишний шаг. Оплата на месте. |
| **Корзина из нескольких услуг** | Усложняет UX на старте. Один визит = одна услуга. |
| **Портфолио «до/после» как сторис** | Дорого в реализации, мастеру нужен контент. Перенесено в SHOULD. |
| **Отзывы с фото** | Нет начальной базы отзывов, нужен модерационный поток. |
| **«Повторить запись»** | Сначала нужен пул возвращающихся клиентов — раньше 100 записей бессмысленно. |
| **Программа лояльности** | После 100+ записей. |
| **Несколько мастеров / выбор мастера** | Витрина одного мастера — он герой. |
| **Поиск по услугам** | Услуг < 20, не нужен. |
| **Deep-link на услугу из канала** | Добавим, когда у мастера будет регулярный контент в TG-канале. |
| **Push-напоминания через бота** | Часть бэка, не TMA. Отдельная задача после релиза. |
| **CloudStorage избранное / последний выбор** | Не критично для первого визита. |
| **Светлая тема / следование `themeChanged`** | Сознательное решение — держим бренд. |
| **Поиск по дате «ближайшее свободное»** | Календарь уже это решает. |
| **Согласие на обработку ПДн / форма email** | Лишние шаги, режут конверсию. ПДн оформим юридически отдельно. |

## Технический минимум для запуска

- HTTPS-хостинг (Cloudflare Pages / Vercel / VPS + nginx).
- Бот зарегистрирован у `@BotFather`, в `/setmenubutton` или inline-кнопка → URL TMA.
- Бэк: эндпоинты `POST /api/auth` (валидация initData), `GET /api/services`, `GET /api/slots?date=`, `POST /api/bookings`, `DELETE /api/bookings/:id`, `GET /api/bookings/mine`.
- БД: 4 таблицы — `services`, `bookings`, `clients` (минимум), `slots` (либо вычислять на лету из расписания мастера).
- Без фронт-фреймворка можно — vanilla JS + CSS-переменные, как остальные страницы проекта. Если будет тесно — Preact (3 КБ) или Svelte.
