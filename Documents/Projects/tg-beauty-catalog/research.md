# Исследование: Telegram Mini App — каталог услуг бьюти-мастера

> Цель: понять, как сделаны существующие TMA в нише бьюти и онлайн-записи, что у них работает, что раздражает, и какие экраны/элементы обязательны для нашего проекта.

---

## 5 примеров для разбора

### 1. Masters (masters-app.ru)

**Что делает.** SaaS для частных бьюти-мастеров и мини-салонов. Запись 24/7, категории (ногти, ресницы, татуаж, парикмахер), профиль клиента с историей и фото, предоплата, галерея работ, аналитика дохода.

**Сильные стороны:**
- Каталог структурирован по нишам — легко сориентироваться.
- Галерея работ — первоклассная сущность (не "файл в чате").
- Встроенная предоплата снижает no-show.
- Уведомления по выбранному каналу клиента (TG/SMS/WA/Max).
- Карточка услуги: фото + длительность + цена + «записаться» одним тапом.

**Слабые стороны:**
- Полноценного TMA нет — Telegram только канал нотификаций, запись идёт через веб-ссылку (лишний клик, теряется нативность).
- Жёсткий шаблон — мастер не может перекрасить под свой бренд.
- Галерея — простая сетка без сторис, без «до/после».
- Нет публичного маркетплейса мастеров — трафик мастер гонит сам.

**Что сделать лучше у себя:**
- Сразу проектировать как TMA, а не сайт + TG-уведомления.
- Дать мастеру индивидуальную палитру (у нас уже есть тёмный премиум + золото).
- Портфолио = первоклассный экран с фильтром по услуге.

### 2. Beauty-App.ru (Frog Studios)

**Что делает.** TMA-витрина услуг + товаров, синхронизированная с YClients. Каталог автообновляется, клиент записывается внутри Telegram, копит бонусы, оставляет отзыв. Демо: `@dar_med_bot`. Подписка от 3 000 ₽/мес.

**Сильные стороны:**
- Бесшовный вход — клиент уже в TG, без регистрации.
- Услуги и товары в одном окне (редкость).
- Программа лояльности с балансом баллов прямо в карточке.
- Автонапоминания через TG вместо SMS — дешевле и читается лучше.

**Слабые стороны:**
- Жёсткая привязка к YClients — без него не работает.
- Подписка + типовой UI «как у всех YClients-виджетов»: утилитарно, без эмоций.
- Фото услуг есть, а полноценной галереи «до/после» по мастеру нет.

**Что сделать лучше у себя:**
- Карточка услуги с историей работ конкретного мастера (а не общим прайсом).
- Эмоциональные фото в премиум-стиле вместо «прайс-таблицы».
- Никакой подписки/привязки к стороннему SaaS — храним данные у себя.

### 3. DIKIDI Online (`@DikidiBot`)

**Что делает.** Маркетплейс-каталог салонов и частных мастеров с геопоиском. ~125 000 компаний, 150 млн записей/год. Выбор услуги → мастера → времени, отзывы с ответом бизнеса, повтор записи в один клик.

**Сильные стороны:**
- **«Повторить запись»** одним тапом — лучший паттерн удержания.
- Отзывы с ответом бизнеса видны до записи.
- Привычная воронка: услуга → мастер → слот (3 шага).
- Прозрачные слоты в виде сетки 30-минутных кнопок.

**Слабые стороны:**
- «Магазин всего» — карточка мастера тонет, нет премиум-ощущения.
- Слабый TMA — основной упор на нативные приложения.
- Отзывы шаблонные («всё ок, спасибо»), без фото работ.
- Перегруженные фильтры в навигации.

**Что сделать лучше у себя:**
- Не строить «каталог всего» — **один мастер герой**: лендинг-первый-экран с фото и историей.
- Отзывы обязательно с фото «до/после», а не текстом.
- Забрать паттерн «Повторить как в прошлый раз» на главной для возвращающихся.

### 4. telegram-salon-booking-bot (open-source, GitHub: nazgool97)

**Что делает.** Self-hosted TMA-шаблон: каталог услуг → выбор мастера → календарь → слот → подтверждение. Корзина бронирований (несколько услуг за один визит), история и предстоящие визиты.

**Сильные стороны:**
- Классический понятный flow в 4 шага.
- Встроен в Telegram — не нужен отдельный логин.
- **Корзина услуг** — редкая, но полезная штука (маникюр + педикюр за один визит).

**Слабые стороны:**
- Дизайн утилитарный, под стандартный Telegram theme.
- Нет витрины мастера как личного бренда — бухгалтерия, а не «вау».
- Фото услуг — заглушки.
- Все салоны, развернувшие шаблон, выглядят одинаково.

**Что сделать лучше у себя:**
- Тот же 4-шаговый wizard, но в премиум-обёртке.
- Корзина из 2–3 услуг за один визит (у SaaS-конкурентов этого почти нет).
- Экраны «история» и «предстоящие» забираем как паттерн.

### 5. Паттерн агентств (BotLabs / OkiBot / dev.family / KIK online)

**Что делают.** Кастомные TMA под клиентов салонов и частных мастеров. Живые ссылки не публикуют, но кейсы и портфолио описывают повторяющиеся решения.

**Сильные стороны:**
- **Витрина-сторис** — портфолио прокручивается как Instagram Stories прямо в TMA.
- **Глубокие ссылки** из постов канала: тап на «новую услугу» → сразу карточка услуги с кнопкой «записаться».
- **Telegram Stars / прямая оплата** для предоплаты слота.
- Напоминания за 24 ч и 2 ч прямо в Telegram.

**Слабые стороны:**
- Однообразные тёмно-белые UI «по референсам Avito».
- Игнорируют нативные элементы Telegram (MainButton, BottomSheet, haptic) — теряется ощущение «части мессенджера».
- Часто грузят все услуги одним списком на 40 позиций — клиент устаёт.

**Что сделать лучше у себя:**
- Использовать **нативный Telegram UI** по максимуму: MainButton, BackButton, haptic, themeParams.
- Услуги — **группами по 3–5 категорий**, у каждой 5–8 позиций.
- Сторис «до/после» прямо в карточке мастера.
- Deep-link на конкретную услугу из постов канала.

## Типовые экраны TMA «каталог + запись»

Стандартный набор — 6–9 экранов, путь до записи **3–5 тапов**.

| # | Экран | Что внутри |
|---|-------|------------|
| 1 | **Главная / Onboarding** | Фото мастера, оффер, USP, приветствие по имени из `initData.user.first_name`, MainButton «Смотреть услуги» |
| 2 | **Каталог услуг** | Категории, карточки с фото-превью, цена «от», длительность |
| 3 | **Карточка услуги** | Фото-результат, описание, цена, длительность, состав, MainButton «Выбрать время» |
| 4 | **Выбор мастера** (если несколько) | Фото, специализация, рейтинг, мини-портфолио |
| 5 | **Календарь записи** | Горизонтальный скролл дней + сетка 30-минутных слотов, свободные/занятые визуально различимы |
| 6 | **Подтверждение / Корзина** | Сводка: услуга + мастер + дата + время + цена; контакт через `requestContact`; MainButton «Подтвердить» |
| 7 | **Оплата / Success** | Telegram Stars / Invoice; экран успеха с haptic `success` |
| 8 | **Мои записи** | Активные + прошедшие, кнопки «Перенести», «Отменить», «Повторить» |
| 9 | **Профиль** | Данные клиента, бонусы, любимые мастера |
| 10 | **Портфолио** | Галерея работ, фильтр по услуге, слайдер ДО/ПОСЛЕ |
| 11 | **Отзывы** | С фото клиента, оценкой, привязкой к услуге, ответом мастера |
| 12 | **О мастере / Сертификаты** | Опыт, образование, лицензии (особенно для косметологии) |
| 13 | **Контакты** | Адрес + карта, телефон, мессенджеры, часы работы |

## Обязательные элементы TMA

### Платформенные (нативность)
- **`WebApp.ready()`** при загрузке — иначе фликер.
- **`WebApp.expand()`** на главной — полный viewport.
- **MainButton** — главный CTA каждого экрана. Поддерживает `showProgress()`, блокирует двойной тап.
- **BackButton** — на всех экранах кроме главной; `hide()` при выходе.
- **ThemeParams + CSS-переменные** (`--tg-theme-bg-color`, `--tg-theme-button-color`, …) — автосинхронизация со светлой/тёмной темой пользователя. Подписка на `themeChanged`. ⚠️ У нас фиксированная палитра «тёмный премиум + золото» — нужно решить: уважаем тему юзера или держим свой бренд (рекомендую второе, но с принудительно тёмным фоном).
- **Viewport** — слушать `viewportChanged`, использовать `viewportStableHeight`, учитывать safe-area iOS.
- **HapticFeedback** — `impactOccurred('light')` на тапах, `notificationOccurred('success'/'error')` на результатах, `selectionChanged()` на выборе слота.
- **`initData`** — валидация HMAC-SHA256 с bot token на бэке + проверка `auth_date`. Авторизация через initData, **не JWT в localStorage** (на iOS WebView изолирован, теряется).
- **CloudStorage** — настройки (избранное, последний мастер), до 1024 ключей.
- **Платежи** — `openInvoice()` для Telegram Stars или внешний провайдер.
- **Popups** — `showConfirm()` для отмены, `showAlert()` для ошибок (вместо `alert()`).

### Контентные (бьюти-специфика)
- **Портфолио ДО/ПОСЛЕ** — слайдер с разделителем, фильтр по услуге. Lazy-load, WebP/AVIF, превью-блюр.
- **Сертификаты и регалии** — отдельный экран, тап → полноэкранный просмотр. Усиливает доверие для косметологии/перманента.
- **Отзывы с фото клиента** (с согласия), оценкой, привязкой к услуге. Автозапрос через 24ч после визита через бот.
- **Прайс** — цена «от», честная длительность, отдельный экран с полным прайсом (для «только посмотреть»), опционально PDF-выгрузка.
- **Карточка мастера** — фото в hero, годы опыта, мини-портфолио.
- **Social proof** — «записалось на этой неделе», «довольных клиентов».
- **Лояльность** — бонусные баллы в профиле, «5-я процедура в подарок», напоминания через бота.

## Best practices и типовые ошибки

### ✅ Делать
- Один экран — одно ключевое действие.
- Skeleton-загрузка вместо спиннеров.
- LCP ≤ 2.5с, INP ≤ 200мс — задержка воспринимается как поломка.
- Кэш каталога в CloudStorage/IndexedDB.
- Тёплое приветствие по имени из `initData`.
- Имя и контакт берём из `initData` — не спрашиваем телефон до подтверждения слота.

### ❌ Ошибки новичков
- Кастомные плавающие кнопки вместо MainButton — теряется доверие и конверсия.
- Светлый фон при тёмной теме Telegram — мгновенный bounce.
- Дублирование навигации (свой «назад» + системный BackButton).
- Хранение токена в localStorage (на iOS теряется).
- Горизонтальные свайпы у края экрана — конфликт со swipe-to-close.
- Игнор валидации initData — дыра в безопасности.
- `alert()`/`confirm()` вместо нативных попапов.
- Все 40 услуг одним списком — клиент тонет.

## Killer-фичи, которые забираем себе

1. **Один мастер герой**, а не «магазин всего» — лендинг-первый-экран с фото и историей. Премиум-палитра из CLAUDE.md выделит на фоне утилитарных Masters / open-source шаблонов.
2. **Карточка услуги = фото-результат + длительность + цена + «записаться»**. Без модальных подменю в 3 уровня.
3. **Услуги группами** — 3–5 категорий (маникюр / педикюр / уход / sos-визит), 5–8 позиций в каждой.
4. **Витрина «до/после» как сторис** — драйвер для постов в Telegram-канале мастера. Этого нет у Masters и шаблонов с GitHub.
5. **Корзина из 2–3 услуг за один визит** — взять у open-source шаблона, в SaaS почти нет.
6. **«Повторить как в прошлый раз»** на главной для возвращающихся — паттерн DIKIDI.
7. **Deep-link на конкретную услугу** из постов канала — конверсия выше «открой бот и поищи».
8. **Telegram Stars / Invoice** для предоплаты + fallback «оплата на месте».
9. **Отзывы с фото** (с согласия), сортировка «с фото» по умолчанию.
10. **Минимум форм** — `initData` + `requestContact` только на шаге подтверждения.

## Экспертная оценка

> Идея: Telegram Mini App — каталог услуг одного бьюти-мастера, премиум-личный-бренд (тёмная палитра + золото), фиксированная тема (не подписываемся на `themeChanged`), запись с предоплатой через Telegram Stars, портфолио «до/после» как сторис.

### 👁 Взгляд UX-дизайнера

**Что обязательно должно быть:**
- **MainButton снизу на каждом шаге** — без неё клиент ищет «куда тыкать», на iOS теряет до 30% конверсии. Текст кнопки меняется по контексту: «Выбрать время» → «Подтвердить» → «Оплатить».
- **3–5 тапов до записи**, не больше. Главная → услуга → слот → подтверждение. Всё, что добавляет шаги, режет конверсию.
- **Skeleton-загрузка** вместо спиннеров — ощущение «уже работает».
- **Haptic feedback** на каждом важном действии — выбор слота, подтверждение, оплата. Это бесплатный «вау» и снижает тревогу «нажалось ли».
- **Свободные слоты визуально отличаются от занятых** — не серый текст «занято», а полностью disabled-вид, не кликается.
- **Цена + длительность в карточке услуги ДО открытия** — клиент не должен «копать», чтобы узнать сколько стоит.

**Типичные ошибки новичков:**
- Кастомная плавающая кнопка вместо MainButton — UI выглядит как «сайт в Telegram», а не «часть Telegram». Снижает доверие.
- Дублирование навигации: свой «← Назад» сверху + системный BackButton — пользователь тыкает не туда, бесится.
- Длинная форма «введите имя, фамилию, телефон, email, дату рождения» перед записью — half клиентов отваливается. У нас имя берём из `initData`, телефон через `requestContact` **только на шаге подтверждения**.
- Модальные окна в 3 уровня вложенности — клиент теряется, swipe-to-close закрывает не то.
- Цена «по запросу» / «уточняйте» — главный убийца конверсии в бьюти.
- Горизонтальные свайпы у края экрана конфликтуют со swipe-to-close TMA — теряются жесты в галерее «до/после».

**Что бесит пользователей (из отзывов на DIKIDI и аналоги):**
- Календарь, который показывает только 1 день вперёд — приходится тыкать «дальше» 5 раз.
- «Запись отправлена, мастер свяжется с вами» — клиент не понимает, записан он или нет. Должно быть **подтверждение слота сразу**, без «перезвонит».
- Невозможность отменить запись из приложения — заставляют писать в личку. Нужна кнопка «Отменить» с `showConfirm()`.
- Реклама других услуг на экране оплаты — режет завершение.
- Загрузка фото портфолио в полном размере без lazy-load — TMA «зависает» на 3 секунды на открытии.

### 📈 Взгляд маркетолога

**Что заставит записаться (драйверы конверсии):**

1. **Социальный proof в первом экране** — «Записалось на этой неделе: 47 человек», «312 отзывов, средняя 4.9». Цифры конкретные, не круглые («4.9», не «5.0»).
2. **Фото-результат конкретной услуги** в карточке — не сток, а реальная работа этого мастера. Триггер «хочу так же».
3. **«До/после» как сторис** — самый сильный визуальный аргумент в бьюти. Должен быть на главной, не закопан в раздел.
4. **Прозрачная цена «от X ₽» + длительность** — отсутствие цены = отсутствие записи.
5. **Слот «сегодня вечером / завтра утром»** на главной — импульс «запишусь, пока думаю».
6. **Отзывы с фото клиентов** (не текстовые «всё ок») — снимают страх первого визита. Сортировка «с фото» по умолчанию.
7. **«Повторить запись»** для возвращающихся — закрывает 80% повторных продаж в один тап.
8. **Telegram Stars для предоплаты** + скидка 5–10% за предоплату — снижает no-show на 40–60% (статистика SaaS-сервисов записи).
9. **Имя в приветствии**: «Анна, добро пожаловать» вместо «Здравствуйте» — конверсия выше на 10–15%.
10. **Дедлайн / дефицит**: «Свободно: 2 слота на эту неделю» — работает, если честно.

**Что повышает LTV (повторные записи):**
- Программа лояльности в профиле: «5-я процедура в подарок», прогресс-бар.
- Напоминание о повторе через бот: «Прошло 3 недели с маникюра, готова записаться?» (мягкое, не раз в неделю).
- Сертификаты на день рождения через Stars.
- Подписка на канал мастера прямо из TMA — клиент в воронке постоянно.

**Что НЕ работает (антипаттерны):**
- Скидка «-50% первая запись» в премиум-сегменте — отпугивает целевую, привлекает «халявщиков».
- Поп-ап «оставьте email для рассылки» при входе — мгновенный bounce.
- Промокоды на главном экране — отвлекают от записи.

### ⚙️ Взгляд Telegram-разработчика

**Что работает хорошо:**
- **`initData` + HMAC-валидация** на бэке — даёт авторизацию без логинов/паролей. Бери `user.id`, `first_name`, `username`. ⚠️ Проверяй `auth_date` (≤ 24ч свежести), иначе дыра.
- **MainButton с `showProgress()`** — нативно блокирует двойной тап во время запроса. Лучше любой кастомной кнопки.
- **CloudStorage** — храни избранное, последнего мастера, корзину. До 1024 ключей по 4KB. Синхронизируется между устройствами юзера автоматически.
- **`requestContact()`** для телефона — нативный диалог, выше согласие чем у формы.
- **`openInvoice()` для Telegram Stars** — оплата без редиректов, в один тап. Подходит для предоплаты слота/депозита.
- **Deep-link `t.me/yourbot/app?startapp=service_123`** — открывает сразу карточку услуги. Используй в постах канала.

**Ограничения и подводные камни:**
- **iOS WebView изолирован** — `localStorage`/`sessionStorage` теряются при перезапуске. JWT-токены только в `CloudStorage` или через initData при каждом запросе.
- **`themeChanged` мы игнорируем** (решение по бренду) — но это значит наша светлая золотая графика должна нормально читаться на любом фоне Telegram при первом фрейме до полной загрузки. Сделать `<meta name="theme-color" content="#080b12">` и фон body тёмным сразу инлайн-стилем.
- **`viewportStableHeight` ≠ `viewportHeight`** — клавиатура на Android сжимает viewport. Фиксированные нижние блоки (MainButton — нативная, но если делаешь свой sticky-блок) считай от `viewportStableHeight`, иначе прыгает.
- **Safe-area на iPhone с челкой** — `env(safe-area-inset-bottom)` обязателен для нижних элементов.
- **Telegram Stars — только для цифровых товаров и услуг.** Бьюти-услуга = услуга, проходит. Но Apple/Google могут зарубить «оплату физического визита» — формулируй как «бронирование слота» / «депозит».
- **Размер бандла** — TMA грузится через мобильный интернет в чате. Цель: < 200 КБ JS gzipped до интерактива. Иначе на 3G клиент закрывает.
- **`WebApp.close()` нельзя вызвать после редиректа** — теряется контекст. Закрытие после оплаты делай через `MainButton` + `WebApp.close()` в обработчике.
- **`popup` API ограничен** — текст до 256 символов, до 3 кнопок. Длинные тексты — рендери на экране.
- **HTTPS обязателен**, self-signed не работает. Для dev — ngrok / Cloudflare Tunnel.

**Что лучше НЕ делать:**
- Не хранить токены в `localStorage` — на iOS теряются.
- Не использовать `alert()` / `confirm()` / `prompt()` — игнорируются или выглядят чужеродно. Только `showAlert`/`showConfirm`/`showPopup`.
- Не отключать pull-to-refresh JS-ом — раздражает.
- Не вешать `onclick` на `<body>` — конфликтует со swipe-жестами Telegram.
- Не делать «свой логин через телефон» — обходишь `initData`, ломаешь безопасность и UX одновременно.
- Не использовать `window.open()` для внешних ссылок — только `WebApp.openLink()` (откроет в браузере) или `WebApp.openTelegramLink()`.
- Не блокируй UI на медленные запросы — используй оптимистичные обновления + откат при ошибке.

### 📋 Общий список рекомендаций (приоритизировано)

**MUST — без этого не запускать:**
1. MainButton + BackButton нативные на каждом экране.
2. Валидация `initData` HMAC-SHA256 на бэке.
3. Карточка услуги: фото-результат + цена + длительность + MainButton «Выбрать время».
4. Календарь с реальной сеткой слотов, мгновенное подтверждение (без «мастер свяжется»).
5. Тёмная палитра фиксированная, `<meta theme-color="#080b12">`, тёмный фон body инлайн.
6. Skeleton-загрузка, lazy-load фото, бандл < 200 КБ gzipped.
7. `requestContact()` только на шаге подтверждения, имя из `initData`.
8. Отмена записи внутри TMA через `showConfirm()`.

**SHOULD — добавить в первый месяц:**
9. Портфолио «до/после» как сторис на главной.
10. Отзывы с фото, сортировка «с фото» по умолчанию.
11. Telegram Stars для предоплаты (с фолбэком «на месте»).
12. CloudStorage: избранное, последний мастер.
13. Haptic на тапах по слотам, подтверждении, оплате.
14. Deep-link на конкретную услугу для постов канала.
15. Социальный proof на главной (записей за неделю, средняя оценка).

**NICE TO HAVE — после первой сотни записей:**
16. «Повторить запись» одним тапом на главной.
17. Программа лояльности (5-я процедура в подарок).
18. Корзина из 2–3 услуг за один визит.
19. Мягкое напоминание о повторной записи через бот.
20. Сертификаты на день рождения через Stars.

**NEVER — антипаттерны:**
- Кастомные плавающие CTA вместо MainButton.
- Скидки -50% «первая запись» в премиум-сегменте.
- Длинные формы регистрации.
- `localStorage` для токенов.
- Цена «по запросу».
- «Мастер свяжется с вами для подтверждения» — только мгновенное подтверждение слота.
